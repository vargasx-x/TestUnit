¿Qué es JUnit?

JUnit es el marco de pruebas unitarias de Java más popular. Un marco de código abierto, se utiliza para escribir y ejecutar pruebas automatizadas repetibles.

Como con cualquier otra cosa, el marco de prueba JUnit ha evolucionado con el tiempo. JUnit 4.0 se lanzó en 2006 y 5.0 se lanzó en 2017. Al momento de escribir este artículo, la última versión es 5.9.1. JUnit 5.x ha abordado muchas de las limitaciones anteriores de JUnit y se ha convertido en el marco de pruebas de unidades de Java más sólido.

¿Por qué es importante para las pruebas unitarias en Java?

JUnit es importante para las pruebas unitarias en Java porque facilita la escritura y ejecución de pruebas automatizadas repetibles, lo que ayuda a detectar y corregir errores temprano en el proceso de desarrollo y a verificar que los cambios en el código no hayan introducido nuevos errores.

Los IDE más comunes, como Eclipse e IntelliJ, ya tendrán instalada la integración de pruebas JUnit de manera predeterminada. Si no usa un IDE y tal vez depende únicamente de un sistema de compilación como Maven o Gradle, la instalación de JUnit 4/5 se maneja a través del archivo pom.xml o build.gradle, respectivamente.

Es importante tener en cuenta que JUnit 5 se dividió en 3 módulos, uno de los cuales es un módulo antiguo que admite la anotación/sintaxis de JUnit 4 y 3. El uso de JUnit 3 es muy bajo en este punto y generalmente solo se ve en proyectos mucho más antiguos.

La función assertTimeout en Java se utiliza para verificar que una operación se complete dentro de un período de tiempo especificado. Esta función es especialmente útil en pruebas unitarias (unit testing) cuando deseas asegurarte de que una parte de tu código se ejecute en un tiempo razonable y no tome demasiado tiempo, lo que podría indicar un problema o una degradación del rendimiento en tu aplicación.

Creamos los métodos, los atributos, los Getters y Setters en el paquete Control

Esta será creada en el paquete "Test"

(1).

import static org.junit.jupiter.api.Assertions.assertTimeout;

Esta línea importa el método assertTimeout de la clase de Assertions de JUnit.

assertTimeout se utiliza para verificar si un bloque de código se ejecuta dentro de un límite de tiempo especificado.

(2). import java.time.Duration;

Esta linea importa la clase Duration del paquete java.time. Duration se utiliza para representar una cantidad de tiempo, como segundo, milisegundos, etc,

(3).

import org.junit.jupiter.api.Test;

Esta línea importa la anotación Test de JUnit. @Test se utiliza para marcar un método como un método de prueba.

(4).

import control.AgregarIngredientes;

Esta linea importa la clase AgregarIngredientes del paquete control. Esto asume que tendremos una clase llamada AgregarIngredientes en el proyecto.
